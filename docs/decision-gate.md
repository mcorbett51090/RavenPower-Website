# Decision Gate: Webflow vs Code Path

This is the most important early step in any website project using the `website-design` plugin.

## The Question to Ask

> "Do you (or someone on your team) want to be able to easily edit and update content on the website **without writing or touching code**?"

### If Yes → Path A: Webflow (Recommended for most small businesses)
- Pros: Client can edit content easily, fast to build, professional results, good component system.
- Cons: Hosting costs, some limitations on very custom functionality.
- Best for: Service businesses, consultants, marketing sites, portfolios.

### If No (or they prefer code) → Path B: Next.js + Tailwind + Vercel
- Pros: Maximum performance, full control, excellent SEO, modern stack, easier long-term ownership.
- Cons: Requires developer for content updates (or you build a CMS on top).
- Best for: Clients who want ownership or have more complex needs.

## Why This Matters

This single decision dramatically changes:
- Which agents are primarily used
- What the deliverables look like
- How the site will be maintained 1–3 years from now
- Technology recommendations
- Pricing and scope discussions

**Never skip or rush this gate.**
